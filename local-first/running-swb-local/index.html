    Running Surface Water Body (SWB) Locally¶

    Warning

    This tutorial is still being aligned with the current backend implementation. Use it as a roadmap, but treat the live API docs and backend code as the source of truth for route names and runtime behavior.

    This tutorial walks you through running the Surface Water Body detection pipeline entirely on your local machine—no Google Earth Engine account required.

    What You'll Learn¶

    • How to enable Local Computation Mode
    • How to query the STAC catalog for input data
    • How to run the SWB pipeline via API
    • How to visualize the results

    Prerequisites¶

    Before starting:

    • CoRE Stack backend installed and running
    • Local mode enabled (COMPUTE_MODE=LOCAL in .env)
    • Server running at http://127.0.0.1:8000
    • Celery worker started

    Understanding the Pipeline¶

    Surface Water Body (SWB) detection identifies water bodies like ponds, tanks, and reservoirs from Land Use/Land Cover (LULC) data.

    How It Works¶

    graph LR
    A[LULC Raster] --> B[Extract Water Pixels]
    B --> C[Vectorize to Polygons]
    C --> D[Intersect with MWS]
    D --> E[GeoJSON Output]
    1. Input: LULC raster (class value 1 = Water)
    2. Process: Extract water pixels and convert to vector polygons
    3. Intersect: Match with Micro-Watershed boundaries
    4. Output: GeoJSON with water body geometries and attributes

    Step-by-Step Guide¶

    Step 1: Verify Local Mode¶

    Check that local mode is enabled:

    # In your .env file
grep COMPUTE_MODE .env

# Should output: COMPUTE_MODE=LOCAL

    If not set, add it:

    echo "COMPUTE_MODE=LOCAL" >> .env

    Restart the server after changing .env.

    Step 2: Prepare Your Request¶

    The SWB API requires these parameters:

    {
  "state": "karnataka",
  "district": "raichur",
  "block": "devadurga",
  "start_year": 2022,
  "end_year": 2023
}

    Available Regions

    Check available states/districts in the Django admin under Geoadmin.

    Step 3: Run the Computation¶

    Using curl¶

    curl -X POST http://127.0.0.1:8000/api/v1/generate_swb/ \
  -H "Content-Type: application/json" \
  -d '{
    "state": "karnataka",
    "district": "raichur",
    "block": "devadurga",
    "start_year": 2022,
    "end_year": 2023
  }'

    Using Python¶

    import requests

url = "http://127.0.0.1:8000/api/v1/generate_swb/"
payload = {
    "state": "karnataka",
    "district": "raichur",
    "block": "devadurga",
    "start_year": 2022,
    "end_year": 2023
}

response = requests.post(url, json=payload)
print(response.status_code)
print(response.json())

    Using Swagger UI¶

    1. Visit http://127.0.0.1:8000/swagger/
    2. Navigate to POST /api/v1/generate_swb/
    3. Click "Try it out"
    4. Enter the JSON payload
    5. Click "Execute"

    Step 4: Monitor Progress¶

    The current backend surface returns an initiation response from the standard SWB handler:

    Success Response: 

    {
  "Success": "Generate swb task initiated"
}

    Error Response: 

    {
  "error": "No STAC items found for karnataka/raichur/devadurga. Ensure data is synced."
}

    Step 5: Verify Output¶

    Check the output file:

    ls -lh media/outputs/swb/

    Inspect the GeoJSON:

    # View first feature
head -100 media/outputs/swb/local_swb_output.geojson

# Count features
jq '.features | length' media/outputs/swb/local_swb_output.geojson

    Visualizing Results¶

    Option 1: GeoJSON.io¶

    1. Open http://geojson.io/
    2. Drag and drop your GeoJSON file
    3. View water bodies on an interactive map

    Option 2: QGIS¶

    1. Open QGIS
    2. Layer > Add Layer > Add Vector Layer
    3. Select your GeoJSON file
    4. Style with blue fill for water bodies

    Option 3: Kepler.gl¶

    1. Visit https://kepler.gl/
    2. Upload your GeoJSON
    3. Customize visualization
    4. Export as image or share

    Behind the Scenes¶

    What Happens When You Call the API¶

    # 1. Router intercepts request
@route_local_compute(local_api_view_func=generate_swb_local)
def generate_swb(request):
    ...

# 2. STAC fetcher downloads data
local_files = download_stac_dependencies(
    pipeline="swb",
    state="karnataka",
    district="raichur",
    block="devadurga"
)

# 3. Local engine processes
output = compute_swb_with_geopandas(local_files)

# 4. Returns result
return Response({"Success": "...", "local_output": output})

    Data Downloaded¶

    The STAC fetcher retrieves:

    • lulc_raster.tif - Land Use/Land Cover raster
    • mws_boundary.geojson - Micro-watershed boundaries
    • block_boundary.geojson - Administrative boundary

    These are cached in media/temp_computations/swb/... for reuse.

    Troubleshooting¶

    "No STAC items found"¶

    The requested area doesn't have pre-computed data in the catalog.

    Solutions:

    • Try a different state/district/block
    • Check available data at https://core-stack.org/stac/
    • Use GEE mode for on-demand processing

    "Memory error"¶

    Large areas may exceed available RAM.

    Solutions:

    • Request smaller areas (single block vs entire district)
    • Close other applications
    • Upgrade RAM or use GEE mode

    "Permission denied"¶

    Output directory not writable.

    Solutions: 

    mkdir -p media/outputs/swb
chmod 755 media/outputs
