# Rebuilding The Docs Tree For V3

The new docs tree was rebuilt around a simple principle:

people should be able to keep moving forward without repeatedly backing out of the site to find the next useful branch.

## What Changed

- `Getting Started` became a bridge section instead of a dump of unrelated setup detail
- installation moved into `Developers`
- public data pages and computation pages now point at each other explicitly
- the developer journey was reordered into:
  install
  understand
  run locally
  integrate with Django
  then add Celery, auth, and publication

## Why This Matters

CoRE Stack serves very different kinds of readers:

- data users
- researchers
- developers
- challenge participants
- stewards and local institutions

The docs are better when each of those readers can move through the repo without losing context.
