    Welcome to CoRE Stack¶

    An open geospatial backend for democratic planning, public data access, and computation pipelines.

    Start in the Browser¶

    You do not need a local install to start exploring CoRE Stack.

    • Public Swagger: https://geoserver.core-stack.org/swagger/
    • Local Swagger after install: http://127.0.0.1:8000/swagger/
    • Main backend route map: nrm_app/urls.py

    Two good first calls:

    curl https://geoserver.core-stack.org/api/v1/get_states/

    curl "https://geoserver.core-stack.org/api/v1/get_districts/29/"

    These auth-free geography endpoints are implemented in geoadmin/api.py and routed through geoadmin/urls.py.

    Use Search Early

    If you are busy, use the search button in the header and jump straight to terms like NDVI, LULC, hydrology, Koraput, GeoServer, gee_account_id, or facilities proximity.

    System Architecture

    What Exists Today¶

    CoRE Stack already exposes three useful surfaces:

    • Auth-free geographic discovery for states, districts, blocks, and proposed blocks in geoadmin/api.py
    • API-key public data access for admin lookups, MWS data, generated layer URLs, reports, and geometries in public_api/api.py
    • Task-submission compute endpoints for SWB, LULC, hydrology, terrain, and related pipelines in computing/urls.py

    The public-data layer URL builder lives in public_api/views.py, and the compute handlers mostly queue background work from computing/api.py.

    Installation Path¶

    The backend already ships with an automated installer in installation/install.sh. It currently automates:

    • Miniconda installation
    • Conda environment creation
    • PostgreSQL setup
    • Apache setup
    • backend clone and deploy directory setup
    • .env generation
    • static collection and migrations

    The current backend install guide still calls out manual prerequisites and follow-up such as RabbitMQ and environment configuration in installation/INSTALLATION.md.

    Quick Start¶

    Option 1: Explore the Live API¶

    Start with live Swagger and the API docs in this repo:

    • Swagger UI
    • Public Endpoints

    Option 2: Install Locally¶

    Use the backend installer, then run the local Django server:

    git clone https://github.com/core-stack-org/core-stack-backend.git
cd core-stack-backend/installation
chmod +x install.sh
./install.sh
