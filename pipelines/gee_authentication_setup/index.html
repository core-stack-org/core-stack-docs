    GEE Authentication Setup Guide¶

    Overview¶

    This document explains how to set up and troubleshoot Google Earth Engine (GEE) authentication for the CoreStack backend application.

    Problem Statement¶

    When running Celery tasks that use Google Earth Engine, you may encounter the following error:

    GEEAccount.DoesNotExist: GEEAccount matching query does not exist.

    This error occurs when: 1. The GEEAccount table in the database is empty 2. The GEE_DEFAULT_ACCOUNT_ID in .env points to a non-existent account 3. The gee_account_id parameter passed to a task is invalid or None

    Solution¶

    Automated Setup (Recommended)¶

    Use the test_gee_auth.py script to automatically test your GEE credentials and set up Django:

    # From WSL (Windows Subsystem for Linux)
cd /mnt/y/core-stack-org/core-stack-backend
/home/amitportal/miniconda3/envs/corestack-backend/bin/python test_gee_auth.py --json /path/to/service-account.json --setup

    Manual Setup¶

    If you prefer to set up manually:

    1. Start Django shell: 

      python manage.py shell

    2. Create GEE Account: 

      from gee_computing.models import GEEAccount
from nrm_app.settings import FERNET_KEY
from cryptography.fernet import Fernet
import json

# Read your service account JSON
with open('/path/to/service-account.json', 'r') as f:
    creds = json.load(f)

# Encrypt credentials
fernet = Fernet(FERNET_KEY)
encrypted = fernet.encrypt(json.dumps(creds).encode())

# Create account
account = GEEAccount.objects.create(
    name='default',
    service_account_email=creds['client_email'],
    credentials_encrypted=encrypted,
)
print(f"Created account with ID: {account.id}")

    3. Update .env file: 

      GEE_DEFAULT_ACCOUNT_ID=<account_id_from_step_2>

    4. Restart Celery worker: 

      celery -A nrm_app worker -l info -Q nrm

    Test Script Reference¶

    test_gee_auth.py¶

    A standalone script that tests GEE authentication and can automatically configure Django.

    Usage¶

    # Interactive mode
python test_gee_auth.py

# Test with JSON file only
python test_gee_auth.py --json /path/to/key.json

# Test and auto-setup Django (recommended)
python test_gee_auth.py --json /path/to/key.json --setup

# Only check Django database status
python test_gee_auth.py --check-only

# Custom account name
python test_gee_auth.py --json /path/to/key.json --setup --account-name my-account

    Command Line Arguments¶

    Argument Short Description
    --json -j Path to service account JSON key file
    --setup -s Automatically set up Django with credentials
    --account-name -n Name for the GEE account (default: "default")
    --check-only -c Only check Django database, skip credential test

    What the Script Does¶

    1. Tests GEE Authentication:
    2. Validates the JSON credentials structure
    3. Creates OAuth2 credentials object
    4. Initializes Earth Engine

    5. Tests basic operations (fetching Delhi boundary, simple calculation)

    6. Sets Up Django (with --setup flag):

    7. Encrypts credentials using FERNET_KEY
    8. Creates GEEAccount record in database

    9. Updates .env file with GEE_DEFAULT_ACCOUNT_ID

    10. Provides Diagnostic Information:

    11. Lists all GEE accounts in database
    12. Shows which account is set as default
    13. Identifies missing or misconfigured accounts

    Code Changes Made¶

    1. utilities/gee_utils.py - Enhanced ee_initialize() function¶

    The function now handles missing accounts gracefully:

    def ee_initialize(account_id=GEE_DEFAULT_ACCOUNT_ID):
    # Use default account if account_id is None or invalid
    if account_id is None:
        account_id = GEE_DEFAULT_ACCOUNT_ID

    try:
        account = GEEAccount.objects.get(pk=account_id)
    except GEEAccount.DoesNotExist:
        # Fall back to default account if the specified account doesn't exist
        if account_id != GEE_DEFAULT_ACCOUNT_ID:
            print(f"GEEAccount with id={account_id} not found, falling back to default account")
            account = GEEAccount.objects.get(pk=GEE_DEFAULT_ACCOUNT_ID)
        else:
            raise

    # ... rest of the function

    This change: - Uses GEE_DEFAULT_ACCOUNT_ID when account_id is None - Falls back to default account if specified account doesn't exist - Provides helpful error messages for debugging

    Troubleshooting¶

    Error: "GEEAccount matching query does not exist"¶

    Cause: No GEE accounts in database or invalid GEE_DEFAULT_ACCOUNT_ID

    Solution: Run the setup script: 

    python test_gee_auth.py --json /path/to/key.json --setup

    Error: "PermissionDenied"¶

    Cause: Service account not registered with Earth Engine

    Solution: 1. Go to https://code.earthengine.google.com/register 2. Register the service account email for Earth Engine access

    Error: "Invalid Credentials"¶

    Cause: JSON key file is corrupted or service account was deleted

    Solution: 1. Go to Google Cloud Console 2. Create a new service account key 3. Download and use the new JSON file

    Error: "Network Error"¶

    Cause: Cannot connect to Google servers

    Solution: 1. Check internet connection 2. Verify firewall allows HTTPS to: - earthengine.googleapis.com - oauth2.googleapis.com

    Security Notes¶

    1. Never commit service account JSON files to version control
    2. Add JSON files to .gitignore: 
      # Service account keys
*.json
!package.json
!package-lock.json
    3. The credentials are encrypted before storing in the database using the FERNET_KEY from your .env file
    4. Delete temporary credential files after setup: 
      rm gee_test_credentials.json
